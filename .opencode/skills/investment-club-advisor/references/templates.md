# 解读 与 投资建议 结构模板

Two structures. Adapt headings to the report; keep the operational parts complete.

## 周报解读结构

File: `notes/my-files/investment-club/周报-MMDD-解读.md`. Canonical worked example: `周报-0621-解读.md`.

```markdown
# 韭俱乐部周报 MMDD · 解读

> 解读日期 YYYY-MM-DD。基于 YY.MM.DD 周度报告。缩写均附中文全称。

## 一句话定性
<整体方向 + 当下最关键的变量,一两句>

## 整体策略
- <策略要点,逐条;必持主线缩写要展开>

## 短期变量(N 条)
1. <每条带一句解释>

## 大A账户(沪深 + 港股通 + QDII)
- 仓位 <X进攻/Y防守/Z现金>(上周 …→ 变化方向)
- 主题 / 已预期 / 未充分预期 <逐条,别漏定调与细项>

## 海外账户(美港日韩)
- 仓位 …;已预期 / 未充分预期 <逐条>

## 关键事件日历(★ = 报告标注高权重)
- <每个 dated 事件都列;公司/缩写全展开>   ← 不要压缩

## 模拟组合(打勾清单,胜率仅 60–70%,勿 all in)
- 大A / 海外 各四档(防守/配置/波段/投机),每个板块带原始批注   ← 不要压缩
- 使用注释(报告原 N 条,含 * = 计划未执行)

## 仍未定 / 待消息
- <搜了仍没定的术语/事件>
```

关键纪律:**事件日历**和**四档清单**是最可操作的部分,必须逐项列全,过去的失败是把它们压成一行。

## 日报(早/晚/盘中)处理

不必整篇套周报模板。写**相对最新周报框架的 delta**:节奏是否印证周度判断、资金流向/板块强弱变化、右侧抱团板块的增减。日报与周报冲突时**以日报为准**。多数情况直接并入当天的投资建议,只有带真新信号才单独存档。

## 投资建议结构

File: `notes/my-files/investment-club/YYYY-MM-DD-<主题>建议.md`。已有范例:`2026-06-25-美股配置与MU财报前建议.md`、`2026-06-26-A股场外基金调仓建议.md`。

```markdown
# YYYY-MM-DD <主题>建议

## 当前持仓与俱乐部路线的距离
<站错边 / 站得不深 / 已对齐 / 过度激进 —— 一句定性 + 理由>

## 四档对照
<把真实持仓塞进 现金管理 / 防守 / 进攻配置 / 进攻波段 / 进攻投机,带粗略权重>

## 具体调整建议
<可执行:具体 ticker/基金、方向(加/减/换/不动)、力度意图。A股 与 港美 两本账,分开,不跨币种加总。>

## 风险提示 / 执行原则
<相关祖训(追高毁一生 / 卖飞永赚 / 心慌就减仓 / 遇事不决走一半)+ 用户收入现实约束;声明最终决策在用户。>
```

写真建议(用户明确要),但每条都要落在 报告 / 持仓 / 框架 之一上;不确定就 flag,别假装确定。考虑用户在职业空窗、收入重要——不要照搬俱乐部激进的投机档。
