## 平台特征
- The Verge 采用**计量型付费墙（metered paywall）**：每月有若干免费额度，配额内文章读全文，配额耗尽后同一文章被墙、只剩开头几段 + 订阅召唤。
- 因此**是否被墙取决于会话/cookie 状态，而非 URL**。同一篇文章这次能读全文、下次可能被墙；不能从 URL 或文章本身静态判断它是不是「付费文章」。
- 普通文章的 eyebrow 只是栏目标签（如 `TRANSPORTATION / REPORT / SCIENCE`），不带订阅徽章；顶/侧栏的 `/subscribe?...` 链接是站点通用导航，**不是**该文付费的标记，别据此误判。
- 订阅入口统一指向 `/subscribe`。

## 有效模式
- 判断是否拿到全文，靠**正文是否完整**，不靠静态标记：正文容器 `article`，正常长文有几十段；若只剩开头几段就接订阅/登录召唤，即被计量墙截断。
- 对 morning-report 这类批量任务：一旦发现某篇被墙/截断，跳过换另一篇免费可读文章，不要硬总结残篇（见 web-access 的「付费墙与截断内容」）。

## 已知陷阱
- 不要因为「专用浏览器之前读到过这篇全文」就假定这次也是全文——配额耗尽后同一 URL 会被墙。发现时间：2026-06-08。
- 不要把导航里的 `/subscribe` 链接或 `SIGN IN` 当作「这篇是付费文」的信号；它们在免费文章页也存在。发现时间：2026-06-08。
- 本次验证：用户给的示例文 `/transportation/909480/...` 在专用浏览器（配额内）读到完整 57 段正文、无墙，说明它不是静态付费文，而是计量墙在额度耗尽时才触发——计量墙被触发后的确切拦截元素/文案本次未复现，待下次遇到时补录。发现时间：2026-06-08。
