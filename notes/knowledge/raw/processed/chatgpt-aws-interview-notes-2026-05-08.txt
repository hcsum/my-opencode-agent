Amazon RDS

SQL vs NoSQL tradeoffs


Scaling patterns


Indexing (you’ve touched this → advantage)

“When would you choose DynamoDB over RDS?”

DynamoDB = massive scale, predictable latency, but strict access patterns

4. Networking (this is where many fail)

Amazon VPC


AWS Security Group


AWS IAM


What they test:


Can you design a secure architecture

Public vs private subnets

Least privilege



5. Scaling & High Availability
Key services:


Elastic Load Balancing


Amazon CloudFront

Can your system survive traffic spikes

Auto scaling

Decoupling

Amazon CloudWatch
