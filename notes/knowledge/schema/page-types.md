# Page Types

## Source Pages

- Location: `notes/knowledge/wiki/sources/`
- Purpose: create a high-fidelity extraction of one raw source, preserving the most reusable claims, evidence, structures, and follow-up links without flattening away important detail.
- Default language: same as the source unless the ingest command explicitly overrides it.
- For updated raw versions of the same source, prefer one maintained source page with clear version or supersession notes instead of duplicating nearly identical source pages.
- Default stance: preserve detail that affects reuse, interpretation, or decision quality, even when that makes the page longer than a typical summary.

## Entity Pages

- Location: `notes/knowledge/wiki/entities/`
- Purpose: maintain durable pages for people, companies, products, places, projects, or other named things.

## Concept Pages

- Location: `notes/knowledge/wiki/concepts/`
- Purpose: maintain topic pages for themes, ideas, methods, issues, or recurring patterns.
- Default language: Chinese, while preserving important original-language terms inline when needed.

## Synthesis Pages

- Location: `notes/knowledge/wiki/syntheses/`
- Purpose: store cross-source analysis, comparisons, hypotheses, and working conclusions.
- Default language: Chinese, while preserving important original-language terms inline when needed.

## Report Pages

- Location: `notes/knowledge/wiki/reports/`
- Purpose: store user-facing deliverables such as memos, briefings, snapshots, or decision support documents.
- Default language: Chinese unless the user explicitly asks for another language.
- Use reports when the source merits a dedicated reusable artifact beyond the source page, such as a question bank, decision bank, operational checklist, or debugging flow.
