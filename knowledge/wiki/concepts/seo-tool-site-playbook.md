# SEO Tool-Site Playbook

## Definition

A source-grounded operating model for building SEO-led tool sites: discover viable keywords, filter by intent and ROI, launch fast with page clusters, then compound results through backlinks and weekly iteration.

## Core Components

### 1. Keyword sourcing

- Start from root terms, sitemap monitoring, and winning-site keyword expansion.
- Treat Toolify, competitor backlinks, user request boards, and product-review complaints as supporting discovery channels.

### 2. Keyword evaluation

- Use `KDRoi = (search volume x CPC) / KD` to prioritize monetizable opportunities.
- Use `KGR = allintitle results / monthly search volume` to estimate supply pressure.
- Inspect SERP intent and page-type fit before trusting KD.
- Reject terms that are brand-constrained, legally risky, giant-dominated, or too isolated to grow into a cluster.

### 3. Page-system design

- Keep one primary keyword per page.
- Build a three-layer loop: function page, landing page, result page.
- Favor long-tail internal pages first, then use accumulated authority to move toward larger terms.
- Prefer SSR and explicit multilingual URL structure when internationalizing.

### 4. Off-page growth

- Judge backlink targets by both authority and real search traffic.
- Optimize for referring-domain growth, not just raw link count.
- Build links to specific landing pages instead of only the homepage.
- Scale acquisition in layers: manual first, then semi-automated, then low-risk automation.

### 5. Operating cadence

- Weekly loop: choose keywords, ship pages, build links, review outcomes.
- Review using indexing, impressions, rankings, and conversions.
- Rework or remove pages that show no improvement after two weeks of iteration.

## Strategic Direction From Source

- Main lane: multilingual content-tool clusters.
- Second engine: small-game keyword clusters.
- Tactical only: trend-chasing fast sites.
- Supporting lane: evergreen micro-tools.
